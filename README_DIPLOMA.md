# README_DIPLOMA

Этот файл нужен не для запуска проекта, а для подготовки текста дипломной работы и демонстрации на защите по проекту `ShingaDIP`.

Используйте его как заготовку. Формулировки можно адаптировать под требования кафедры, методички и замечания научного руководителя.

## Что это за проект в формулировке для диплома

`ShingaDIP` можно описывать как локальный демонстрационный прототип интеллектуальной системы поддержки бухгалтерского учета и внутреннего аудита.

Ключевая идея проекта:

- загрузить бухгалтерские операции из `CSV` или `XLSX`;
- загрузить первичные документы в формате `PDF`, `PNG`, `JPG`, `JPEG`;
- извлечь реквизиты документа;
- сопоставить операции и документы;
- выполнить проверку по правилам и анализ аномалий;
- рассчитать риск-балл и статус `OK / WARNING / RISK`;
- сформировать интерпретацию результата в виде аудиторского комментария;
- показать итоговые аналитические отчеты в веб-интерфейсе.

Использование технологий искусственного интеллекта позволяет автоматизировать предварительный анализ данных, повысить наглядность результатов проверки и сократить время на первичную оценку рисков.

## Как правильно описать роль ИИ

Это важный момент для диплома. Лучше писать так:

- система сама рассчитывает признаки риска, риск-балл и статус;
- языковая модель не принимает решение о риске;
- языковая модель интерпретирует уже рассчитанные факты и переводит их в деловой аудиторский комментарий;
- это уменьшает вероятность галлюцинаций и делает поведение системы стабильнее.

Удобная формулировка:

В прототипе реализована трехуровневая схема формирования результата. На первом уровне выполняется машинный анализ на основе правил, сопоставления с документами и модели аномалий. На втором уровне языковая модель формирует интерпретацию результата в профессиональном деловом стиле. На третьем уровне система строит итоговое аудиторское заключение и аналитические отчеты для пользователя.

## Как описать расчет риск-баллов

Если в дипломе есть раздел про методику оценки риска, можно писать так:

1. Для каждой операции рассчитываются признаки отклонений.
2. Каждому признаку присваивается вес.
3. Баллы суммируются с учетом комбинации факторов.
4. Дополнительно учитывается результат модели аномалий.
5. По итоговому баллу присваивается статус:

- `0–29` — `OK`
- `30–69` — `WARNING`
- `70+` — `RISK`

При этом отсутствие документов не должно автоматически ухудшать все результаты, если пользователь вообще не загружал документы. Такой подход позволяет снизить число ложных предупреждений и повысить реалистичность итогового анализа.

## Как описать улучшенную логику AI-отчета

Это можно выделить как отдельный сильный результат работы.

Готовая формулировка:

В доработанной версии проекта AI-отчет формируется не из произвольного свободного запроса к модели, а на основе структурированного машинного слоя, включающего статус операции, риск-балл, признаки отклонений, приоритет проверки, уверенность оценки, категорию риска и ведущие факторы. Языковая модель использует эти данные только для интерпретации, формируя краткий вывод, подробный аудиторский комментарий и рекомендуемое действие. Такой подход делает результат более последовательным, объяснимым и пригодным для демонстрации в учебных целях.

## Какие акценты делать, чтобы диплом выглядел сильнее

- Подчеркивайте, что это не “чат-бот для бухгалтерии”, а прототип системы поддержки принятия решений.
- Делайте акцент на объяснимости: статус операции рассчитывается детерминированно, а AI только интерпретирует результат.
- Показывайте, что у системы есть fallback-логика при недоступности моделей.
- Отдельно упоминайте расширенную отчетность: реестр рисков, причины отклонений, контрагентов, сверку документов, итоговое заключение.
- Если показываете большой набор данных, подчеркивайте, что после калибровки правил удалось снизить число ложных срабатываний.

## Что лучше не писать

- Не пишите, что система полностью заменяет бухгалтера или аудитора.
- Не пишите, что AI “сам принимает финансовые решения”.
- Не используйте формулировки про промышленную готовность, если проект учебный.
- Не обещайте высокую юридическую значимость результата.
