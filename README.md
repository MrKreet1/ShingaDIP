# ShingaDIP

Прототип интеллектуальной системы анализа бухгалтерских операций и первичных документов для автоматизации учета и аудита.

## Что реализовано

- загрузка операций из `CSV` и `XLSX`;
- загрузка первичных документов в формате `PDF`, `PNG`, `JPG`, `JPEG`;
- извлечение реквизитов из документов:
  - `PDF` с текстовым слоем через `pypdf`;
  - изображения через `pytesseract`, если локально установлен движок Tesseract OCR;
- нормализация и валидация бухгалтерских данных;
- набор правил контроля:
  - пустые обязательные поля;
  - дубликаты номера документа;
  - несоответствие сумм, дат и контрагентов;
  - подозрительные описания;
  - нетипичные контрагенты;
  - операции без подтверждающего документа;
- поиск аномалий через `IsolationForest`;
- генерация комментариев и рекомендаций:
  - шаблонно, офлайн;
  - опционально через локальный `LM Studio`;
- визуализация результатов в `Streamlit`;
- экспорт итогового отчета в `CSV` и `JSON`.

## Структура

- `app.py` — веб-интерфейс Streamlit.
- `shingadip/` — ядро прототипа.
- `sample_data/` — демонстрационный набор данных.
- `scripts/generate_demo_documents.py` — генератор простых PDF-документов для демо.
- `tests/` — базовые автоматические проверки.

## Запуск

1. Установите зависимости:

```powershell
pip install -r requirements.txt
```

2. При необходимости сгенерируйте демонстрационные документы:

```powershell
python scripts/generate_demo_documents.py
```

3. Запустите приложение:

```powershell
streamlit run app.py
```

## Локальная LLM через LM Studio

Приложение может отправлять запросы в совместимый OpenAI API endpoint, например:

- `http://127.0.0.1:8080/v1/chat/completions`

### Подключение LM Studio

1. В `LM Studio` откройте `Developer` -> `Local Server`.
2. Убедитесь, что сервер запущен и доступен, например по адресу `http://127.0.0.1:8080`.
3. Загрузите нужную модель кнопкой `Load`.
4. В приложении включите опцию `Использовать локальную LLM через LM Studio`.
5. Оставьте endpoint `http://127.0.0.1:8080/v1/chat/completions`.
6. Если сервер доступен, приложение само покажет список загруженных моделей в выпадающем списке.

Если локальная модель недоступна, прототип автоматически использует офлайн-шаблоны пояснений.

## OCR для изображений

Для извлечения текста из изображений нужен установленный Tesseract OCR. Если его нет, прототип:

- не падает;
- показывает предупреждение;
- продолжает анализ на основе табличных данных и доступных документов.

## Демонстрационный сценарий

По умолчанию интерфейс может использовать:

- `sample_data/operations_demo.csv`
- документы из `sample_data/documents/`

В набор уже заложены типовые отклонения:

- дубликат номера документа;
- завышенная сумма;
- пропуск описания;
- несоответствие суммы документу;
- несоответствие даты документу;
- отсутствие подтверждающего документа.
